Memory model
memory layout
Stack — local vars, return addrs, grows down. ~8KB/thread default
Heap — malloc/free, grows up. Manual management required
.data — initialized globals: static int x = 10
.bss — zero-init globals: static int x; no space in binary
.text — read-only code segment
memory bugs quick ref
Stack overflow — canary value at bottom, watermark pattern
Heap fragmentation — fixed-size memory pools eliminate it
Buffer overflow — use strncpy, snprintf; detect with ASan
Use after free — set ptr = NULL after free(). ASan catches it
Memory leak — goto cleanup in C; RAII / unique_ptr in C++
C / C++ essentials
pointer rules
int *p — pointer to int (data mutable)
const int *p — pointer to const int (data immutable)
int * const p — const pointer to int (pointer immutable)
volatile — prevents compiler optimizing away MMIO reads/writes
volatile ≠ atomic — no CPU reordering protection. Need barriers
/* bit manipulation pattern */ reg |= (1 << 3); // set bit 3 reg &= ~(1 << 3); // clear bit 3 reg ^= (1 << 3); // toggle bit 3 val = (reg >> 3) & 0x7; // extract [5:3]
C++ smart pointers + RAII
unique_ptr — sole ownership, no copy, move-only. Use by default
shared_ptr — shared ownership, ref counted. Use when shared
weak_ptr — observe shared_ptr without owning. Break cycles
RAII — acquire in ctor, release in dtor. No leaks on exception
auto p = std::make_unique<uint8_t[]>(64); // freed automatically on scope exit // even if exception is thrown class Lock { Lock() { mutex_acquire(); } ~Lock() { mutex_release(); } // RAII };
Protocol cheat sheet
I2C key facts
2 wires — SCL clock, SDA data. Open-drain + pull-up resistors
Speeds — 100kHz / 400kHz / 1MHz / 3.4MHz HS mode
ACK — receiver pulls SDA low after each byte. NACK = high
NACK causes — wrong addr, device busy, bad register, power issue
Bus stuck low — send 9 clock pulses to recover (i2c_recover_bus)
Repeated START — write reg addr then read without releasing bus
i2cdetect -y 1 # find all devices i2cdump -y 1 0x40 # dump all registers i2cget -y 1 0x40 0x8B w # read word i2cset -y 1 0x40 0x01 0x80 # write
PCIe key facts
LTSSM — Detect → Polling → Config → L0 (active link)
PERST# — reset signal, must deassert before link can form
REFCLK — 100MHz reference clock, required for link training
BAR — base address register, OS allocates memory space
AER — advanced error reporting: correctable vs uncorrectable
MSI-X — message signaled interrupts, preferred over legacy INTx
lspci -vvv # enumerate all devices dmesg | grep -i pcie # link state messages dmesg | grep -i aer # error events setpci -s 0:1.0 COMMAND=0x06 # enable device
SPI key facts
4 wires — SCLK, MOSI, MISO, CS. Full duplex
CPOL — clock polarity. 0 = idle low, 1 = idle high
CPHA — clock phase. 0 = sample on 1st edge, 1 = 2nd edge
Mode 0 — CPOL=0, CPHA=0. Most common
CS timing — CS must assert before and deassert after transfer
No ACK — no built-in acknowledgment, master drives all timing
SMBus / PMBus key facts
SMBus — subset of I2C with timeouts, PEC, fixed voltage levels
PMBus — power management protocol built on SMBus
READ_VOUT 0x8B — read output voltage (linear11 encoding)
VOUT_COMMAND 0x21 — set target output voltage
STATUS_WORD 0x79 — fault summary register, check first
OPERATION 0x01 — turn rail on/off
Debug checklists
PCIe not enumerating
1Power rail — PCIe power present and stable?
2PERST# — reset deasserted correctly?
3REFCLK — 100MHz reference clock present?
4LTSSM — stuck in Detect or Polling?
5Config space — can you read Vendor/Device ID?
6BAR — did OS allocate address space?
7Driver probe — did probe() get called?
8AER logs — correctable/uncorrectable errors in dmesg?
boot hang — no UART
1Power rails — all required rails present and sequenced?
2Reset — RESET_N deasserted at correct time?
3Clock — boot clock source present and locked?
4Boot ROM — executing? Attach JTAG to verify
5BL1 — flash accessible? CRC valid?
6DDR init — training passing? Silent hang common here
7UART init — baud rate and pinmux before first print?
8Watchdog — disabled or kicking during bring-up?
DMA corruption
1Cache clean before DMA write (CPU → device)
2Cache invalidate before DMA read (device → CPU)
3Write barrier before ringing the doorbell
4Read barrier after completion flag
5Descriptor ownership bit set correctly before submit?
6Buffer alignment meets DMA requirements?
7Coherent vs non-coherent — correct mapping type?
system resets randomly
1Reset reason register — WDT, panic, thermal, external?
2PMIC logs — power droop or overcurrent event?
3Watchdog — is WDT being kicked properly?
4Kernel panic — check retention memory for last oops
5Thermal — hitting throttle or shutdown threshold?
6External reset — RESET_N being asserted externally?
7Reproduce under load — correlate with activity type
